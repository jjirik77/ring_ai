CIA Triad

The CIA Triad

The fundamental model for information security. Every security control targets at least one of these three pillars.

Confidentiality

Ensuring that data is hidden from unauthorized users and only accessible to those with a valid "need to know."

"If a secret is leaked, Confidentiality is dead."
  • Encryption: Protecting data at rest (AES) and in transit (TLS).
  • Access Control: Implementing MFA and Permission sets (RBAC).
  • Least Privilege: Granting only the minimum access needed for a job.
  • Obfuscation: Making code or data difficult to read (Steganography).

Integrity

Assurance that data is accurate, complete, and has not been modified by unauthorized parties.

"If the data changes without permission, Integrity is gone."
  • Hashing: Generating unique fingerprints (SHA-256) to detect tampering.
  • Digital Signatures: Combining hashing and encryption to prove origin.
  • Non-repudiation: Ensuring a sender cannot deny sending a message.
  • Versioning: Tracking changes to restore correct data states.

Availability

The guarantee that systems and data are accessible to authorized users whenever they are needed.

"If the user can't log in, Availability has failed."
  • Redundancy: Using RAID, Load Balancers, and Clusters.
  • DDoS Mitigation: Filtering malicious traffic bursts.
  • Patch Management: Fixing bugs that could cause crashes.
  • SLA Tracking: Measuring "Uptime" percentages (e.g., 99.99%).

Key Acronyms to Know

DRP Disaster Recovery Plan: Primary focus is Availability.
HMAC Hash-based Message Auth Code: Focused on Integrity.
RBAC Role-Based Access Control: Focused on Confidentiality.
